# План работ по оценке ИИ-агентов в разработке ПО

## 1. Цели пилота

Основные:
- ускорение разработки (рост пропускной способности команды)  
- повышение качества кода  
- автоматизация рутинных задач  

Ключевые метрики:
- время выполнения задач  
- количество багов после релиза  
- доля правок после ревью  
- время на написание тестов  
- время на boilerplate  

---

## 2. Область оценки (Scope)

Типы задач:
- разработка нового кода  
- доработка существующего  
- написание тестов  
- QA-автоматизация  
- интеграции / API / SQL / скрипты  

Контекст:
- полный контекст проекта  
- без RAG (этап 1)  
- с RAG (этап 2, опционально)  

---

## 3. Кандидаты

Основные:
- Qwen Code (qwen3.5-coder, on-prem)  
- Continue  

Альтернативные:
- Code Llama / DeepSeek Coder  
- Tabby (self-hosted)  
- Open Interpreter  

---

## 4. Дизайн эксперимента

Формат:
- один разработчик выполняет задачи с ИИ и без ИИ по очереди  

Участники:
- 6–10 разработчиков  

Срок:
- 2–3 месяца  

Разбиение задач:
- тип задачи (новый код / доработка / тесты / QA)  
- сложность (low / medium / high)  
- использование ИИ (да/нет)  
- используемый агент  

---

## 5. Методика измерений
[Метрики](METRICS.md)

Скорость:
- фиксируется время начала и окончания задачи  
- фиксируется активное время разработки  

Качество:
- количество багов на задачу  
- при необходимости — уровень критичности  

Code review:
- количество замечаний  
- доля изменений после ревью  

Автоматизация:
- время на написание тестов  
- время на boilerplate  
- доля сгенерированного кода (оценочно)  

---

## 6. Контроль рисков

Галлюцинации:
- фиксация случаев использования несуществующих API  
- фиксация логических ошибок  

Безопасность:
- применение линтеров  
- статический анализ  
- проверка зависимостей  

Дополнительно:
- доля кода, отклонённого из-за ошибок ИИ  

---

## 7. Оценка стоимости владения (TCO)

Инфраструктура:
- требования к CPU/GPU  
- требования к памяти  
- задержки  

Эксплуатация:
- сложность развёртывания  
- обновление моделей  
- поддержка  

Инженерные затраты:
- интеграция (IDE, CLI, пайплайны)  
- разработка RAG (будущий этап)  

---

## 8. Этапы работ

Этап 1. Подготовка (1–2 недели):
- выбор агентов  
- развёртывание в on-prem среде  
- настройка логирования  
- подготовка шаблонов метрик  

Этап 2. Пилот (6–10 недель):
- выполнение задач разработчиками  
- сбор метрик  
- мониторинг процесса  

Этап 3. Анализ (2 недели):
- сравнение агентов  
- анализ результатов  
- выявление рисков  

Этап 4. Принятие решения:
- рассмотрение архитектурным комитетом  
- согласование с ИБ  
- утверждение руководством  

---

## 9. Критерии принятия решения

Минимальные условия:
- отсутствие ухудшения качества  
- отсутствие критических рисков  

Целевые показатели:
- ускорение разработки не менее 15–30%  
- снижение времени на тесты и boilerplate  
- отсутствие роста количества багов  

---

## 10. Формат итогового отчёта

- сравнение агентов  
- метрики (таблицы, графики)  
- успешные и проблемные кейсы  
- выявленные риски  
- оценка TCO  
- рекомендация по внедрению
